📦
Változó Vlad
VARIABLE  ·  int, char, float...

Egy doboz, amibe pontosan egy érték fér be egyszerre. A doboznak van neve, típusa, és ha új értéket teszel bele, a régi nyomtalanul eltűnik. Vlad folyamatosan identity crisisben van.

"Ma 7 vagyok. Holnap talán 'kutya'. Miért ítélsz."
int vlad = 7; // Vlad = 7 vlad = 42; // Vlad most 42. Viszlát 7. char nev[] = "kutya"; // Vlad karaktersorozat float pi = 3.14f; // Vlad tizedestört
⚠ CLASSIC TRAP
Inicializálatlan Vlad értéke random szemét. Ne feltételezd, hogy 0!
👉
Pointer Pete
POINTER  ·  int*, char*, void*

Pete nem mondja meg mi ő — csak megmutatja hol van a valódi érték. Van egy memóriacíme, és ha azt követed: ott találod az adatot. Ha Pete érvénytelen helyre mutat → Segfault Samu megjelenik.

"Én nem vagyok az érték. Én tudom, hol van."
int ertek = 99; int* pete = &ertek; // Pete = cím printf("%p\n", pete); // a cím printf("%d\n", *pete); // 99 (dereference) *pete = 100; // ertek most 100
⚠ HALÁL
int* pete = nullptr; → *pete → Segfault Samu jön el.
🪞
Xerox Chad
PASS BY VALUE  ·  Érték szerinti átadás

Amikor egy függvény kap valamit, másolatot kap, nem az eredetit. A másolat szenvedhet, változhat, meghalhat — Xerox Chad az eredeti, és ő mit sem érez.

"Küldtem egy másolatot. Az eredeti én vagyok."
void doNothing(int x) { x = 999; // csak a másolat változik! } int chad = 5; doNothing(chad); // chad még mindig 5 printf("%d", chad); // → 5
💡 MEGJEGYEZNI
Nagy struktúráknál a másolás lassú. Érdemes lehet pointert vagy referenciát átadni.
🔑
Key Dealer Kevin
PASS BY REFERENCE  ·  int&

Kevin odaadja a lakáskulcsait. Bárki, aki megkapja, beléphet és átrendezheti a lakást. Kevin hazaér, és minden változás valós. Nincs másolat. Teljes hozzáférés — vagy teljes káosz.

"A kulcs nálad. Az én lakásom."
void megduplaz(int& x) { // C++ ref x = x * 2; // az EREDETI változik } int kevin = 10; megduplaz(kevin); // kevin most 20
⚠ C-BEN NINCS REFERENCIA!
C-ben ugyanezt pointerrel éred el: void f(int* x) { *x *= 2; }
🚧
Scope Sensei
SCOPE / HATÓKÖR  ·  { }

Sensei csak a saját blokkjában él. A { jelzi a születését, a } a halálát. Kívülről hívni őt: lefordíthatatlan bűn. Ő tudja ezt. Elfogadta.

"A falaimon belül minden az enyém. Kívül? Nem létezem."
{ int sensei = 99; // él printf("%d", sensei); // ✓ } // sensei meghalt printf("%d", sensei); // ✗ error: undeclared identifier
💡 SCOPE TÍPUSOK
Lokális (függvény), Globális (fájl), Block ({ }), Függvény-paraméter
🫥
Generic Greg
TEMPLATE / SABLON  ·  template<T>

Greg bármivé válhat, de csak akkor dől el mi lesz belőle, amikor ténylegesen használják. Addig ő csak <T> — egy típus nélküli, lebegő egzisztencia a fordítás előtt.

"Mondd meg ki legyek, és azzá leszek. De csak egyszer."
template<typename T> T osszeAdd(T a, T b) { return a + b; } // Használat: osszeAdd<int>(3, 4); // → 7 osszeAdd<float>(1.5, 2.5); // → 4.0
💡 C++ ONLY
Template csak C++-ban létezik. C-ben ezt makróval vagy void*-gal oldják meg.
🌀
Recursion Rick
REKURZIÓ  ·  önhívás + base case

Rick mindig saját magát hívja segítségül a problémájához. Ha van kilépési feltétel (base case): megoldódik. Ha nincs → Stack Overflow Stan érkezik és az egész verem összeomlik.

"Ha te se tudod, kérdezd meg Ricket. Rick kérdezze meg Ricket."
int faktorialis(int n) { if (n == 0) return 1; // base case! return n * faktorialis(n-1); } // f(3) = 3 * f(2) = 3 * 2 * f(1) // = 3 * 2 * 1 * f(0) = 6
⚠ STACK OVERFLOW
Base case nélkül Rick végtelen mélységbe zuhan. Stack mérete véges.
⚰️
Null Nick
NULL / NULLPTR  ·  a nagy semmi

Nem nulla, nem üres string — semmi. Nick sehova sem mutat. Ha hozzányúlsz: Segfault Samu jön el személyesen. Ha ellenőrzöd mielőtt dereferenceled: élsz.

"Én nem vagyok. Mégis mindenhol jelen vagyok."
int* nick = nullptr; // C++ int* nick = NULL; // C // Helyes ellenőrzés: if (nick != nullptr) { printf("%d", *nick); // biztonságos }
⚠ LEGTÖBB BUG OKA
A legtöbb C/C++ crash mögött Null Nick rejtőzik. Mindig ellenőrizd!
⚠ BOSS KARAKTEREK — HA EZEKET LÁTOD, BŐVÍTENI KELL

💀 SEGFAULT SAMU

Akkor jelenik meg, ha érvénytelen memóriacímet ér el a programod. Null Nick dereferencelése, wild pointer, stack overflow — mind az ő munkája. A program azonnal megszűnik. Nem lehet tárgyalni vele.

Segmentation fault (core dumped)
🌊 STACK OVERFLOW STAN

Recursion Rick hívja elő, ha elfelejt megállni. A call stack véges méretű. Ha betelik, Stan gondoskodik arról, hogy a program emlékezetes véget érjen. Base case nélkül Rick mindig Stant hívja.

Stack overflow / Stack smashing detected
👻 MEMORY LEAK MIKI

Ha malloc() / new után elfelejtesz free() / delete-et hívni, Miki csendesen ellopja a memóriád. A program fut, de lassul. Napokig nem veszed észre.

valgrind: definitely lost: 2,048 bytes
🎲 UNDEFINED BEHAVIOR UBUL

A legelborultabb boss. Ha meghívod (pl. signed integer overflow, array out of bounds, inicializálatlan változó olvasása) — bármi történhet. Ma működik, holnap nem, más gépen más. Nincs szabály.
