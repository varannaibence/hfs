📐
Osztály Olga
CLASS  ·  class Valami { };

Olga egy tervrajz. Leírja, milyen lesz egy dolog — milyen adatai lesznek és mit tud csinálni — de ő maga nem hoz létre semmit. Amíg nem példányosítják, csak egy papírlap.

"Én a terv vagyok. Nem a ház."
class Kutya { public: std::string nev; int kor; void ugat() { // adatok + viselkedés } }; // ← pontosvessző kötelező!
💡 ALAP
Az osztály csak leírás. Memóriát nem foglal magától.
🐕
Objektum Oszkár
OBJECT / INSTANCE  ·  Kutya k;

Oszkár az, aki ténylegesen létezik. Ő az Osztály Olga tervrajzából életre hívott, saját memóriával rendelkező konkrét példány. Több Oszkár is lehet egyszerre — mind más, mind Kutya.

"Olga a tervem. Én a valóság."
Kutya oscar; // stack-en oscar.nev = "Oszkár"; oscar.kor = 3; oscar.ugat(); Kutya* buddy = new Kutya(); // heap-en delete buddy; // te takarítasz
💡 STACK VS HEAP
Stack: automatikusan megszűnik. Heap: new után neked kell delete.
🍼
Konstruktor Kristóf
CONSTRUCTOR  ·  OsztályNev()

Kristóf az a srác, aki minden objektum születésekor automatikusan fut le. Beállítja az értékeket, lefoglalja az erőforrásokat. Nem hívod te — ő jön magától, amikor az objektum megszületik.

"Senki nem kéri, hogy legyek ott. Mégis ott vagyok."
class Kutya { public: std::string nev; // konstruktor: Kutya(std::string n) { nev = n; // Kristóf beállít } // default konstruktor: Kutya() : nev("Névtelen") {} };
💡 OVERLOAD
Lehet több konstruktor is — különböző paraméterlistával.
🧹
Destruktor Dezső
DESTRUCTOR  ·  ~OsztályNev()

Dezső utolsónak megy el. Amikor egy objektum meghal (scope vége, delete), Dezső automatikusan lefut és elpakolja, amit Kristóf hozott létre. Ha Dezső nem takarít el → Memory Leak Miki jön.

"Mindenki elmegy. Én maradok — hogy letakarítson."
class Kutya { public: int* adat; Kutya() { adat = new int(42); } ~Kutya() { // ← Dezső delete adat; // felszabadítás! } };
⚠ SZABÁLY OF THREE
Ha van destruktored → kell copy constructor + copy assignment is!
🚪
Bezáró Bence
ENCAPSULATION  ·  public / protected / private

Bence a bouncer. Ő dönti el, ki láthat bele az osztályba és ki nem. Háromféle ajtója van — attól függ ki mehet be.

"A listán vagy, vagy nem vagy. Más nincs."
KulcsszóSaját osztályLeszármazottMindenki
public ✅✅✅
protected ✅✅❌
private ✅❌❌
class Bank { public: void befizet(int x) { egyenleg += x; } protected: int hitelkeret = 10000; private: int egyenleg = 0; // SENKI nem látja };
🧬
Öröklődő Olivér
INHERITANCE  ·  class B : public A

Olivér megörökli a szülei tulajdonságait — és ráadásul saját dolgokat is hozzáad. A szülőosztály mindent megad, a gyerek kibővítheti vagy felülírhatja. Olivérből akár több szint is lehet.

"Apa mindent adott. Én még többet tettem hozzá."
class Allat { // szülő public: void lelegzik() { } }; class Kutya : public Allat { public: // Kutya IS-A Allat void ugat() { } // + saját metódus }; // Kutya tud lelegzik()-et is!
💡 IS-A SZABÁLY
Öröklés = "IS-A" kapcsolat. Kutya IS-A Állat ✅  |  Kutya IS-A Autó ❌
🎭
Polimorf Péter
POLYMORPHISM  ·  virtual + override

Péter különböző arcokat visel, de mindig ugyanolyan névvel szólítják. Ugyanaz a függvénynév (hangotAd()), de Kutyán, Macskán, Madáron hívva mindenki mást csinál. A maszk mögött az igazi típus dönt.

"Mindenki hangotAd()-nak hív. De senki nem tudja, mit fogok mondani."
class Allat { public: virtual void hangotAd() { // alap } }; class Kutya : public Allat { void hangotAd() override { // "Vau!" } }; class Macska : public Allat { void hangotAd() override { // "Miau!" } };
⏳
Late Binding Lajos
DYNAMIC DISPATCH  ·  virtual, futásidő

Lajos nem dönt előre. A fordító nem tudja, melyik hangotAd() fog lefutni — ezt csak futásidőben dönti el a vtable. Rugalmas, de lassabb mint Early Eddie. virtual kulcsszó kapcsolja be.

"Ma még nem tudom, mit fogok mondani. Majd meglátjuk futáskor."
Allat* a = new Kutya(); // Melyik hangotAd() fut le? // A fordító NEM tudja... a->hangotAd(); // ...futáskor derül ki: Kutya! // → vtable pointer dönti el
💡 VTABLE
A virtual egy rejtett mutatótáblát (vtable) hoz létre az osztályban.
⚡
Early Binding Eddie
STATIC DISPATCH  ·  fordítási idő

Eddie már fordítási időben tudja, melyik függvény fog lefutni. Nem kell keresgélni futáskor — a fordító direkt beledrótozza a hívást. Gyorsabb mint Lajos, de nem tud "meglepetés típust" kezelni.

"Én már a fordítóval megbeszéltük. Futáskor nincs megbeszélnivaló."
class Kutya { public: // NEM virtual → Eddie dönt void hangotAd() { // "Vau" } }; Kutya k; k.hangotAd(); // fordítókor kötve // Template is early binding: osszeAdd<int>(1,2); // fordítókor
💡 SEBESSÉG
Eddie általában gyorsabb. Lajos rugalmasabb. Polimorfizmushoz Lajos kell.
👻
Absztrakt Anikó
ABSTRACT CLASS  ·  pure virtual

Anikó ígér mindent, de nem csinál semmit. Van legalább egy pure virtual függvénye (= 0), ami miatt soha nem lehet belőle közvetlen objektumot csinálni. Leszármazottjai kötelesek megvalósítani.

"Tudom, mit kell csinálni. De én azt nem csinálom meg. Ez a te dolgod."
class Allat { // absztrakt! public: virtual void hangotAd() = 0; // ^^ pure virtual = 0 void lelegzik() { } // ← ez lehet }; Allat a; // ✗ HIBA! Nem lehet! Kutya k; // ✓ ha Kutya impl.
💡 MIKOR?
Ha van közös alap viselkedés + kötelező metódusok → absztrakt osztály.
📋
Interfész István
INTERFACE  ·  tiszta absztrakt osztály

István egy szerződés. Semmi implementáció, csak ígéret: "aki velem dolgozik, megvalósítja ezeket a metódusokat." C++-ban nincs külön interface kulcsszó — tisztán absztrakt osztályként valósítják meg.

"Nem mondok el semmit rólad. Csak azt, mit KELL tudnod."
class IHangos { // interfész public: virtual void hangotAd() = 0; virtual ~IHangos() = default; }; class Kutya : public IHangos { // kötelező impl. void hangotAd() override { } };
⚠ ABSZTRAKT vs INTERFÉSZ
Absztrakt: van közös kód + pure virtual. Interfész: CSAK pure virtual, semmi más.
⚗️
Példányosító Pál
INSTANTIATION  ·  Osztaly obj;

Pál az a pillanat, amikor az osztályból objektum lesz. Memóriafoglalás + konstruktor hívás — ez a születés. Stacken vagy heapen történhet, és azonnal Kristóf fut le.

"Olga adja a tervet. Én hozom létre. Kristóf köszönt."
// Stack (automatikus) Kutya k1("Rex"); // Pál + Kristóf // Heap (manuális) Kutya* k2 = new Kutya("Blöki"); delete k2; // Dezső fut! // Tömb Kutya csapat[3]; // 3x Kristóf
💡 SORREND
Születés: Pál ⟶ Kristóf. Halál: Dezső ⟶ felszabadítás.
